# P3.P1 — Interceptor (JSON Syntax Repair) Contract

## Scope
This contract defines a deterministic, headless “repair layer” for malformed JSON payloads that are intended to represent tool/function arguments.

Non-goals:
- No network/provider calls.
- No streaming.
- No provider-layer changes.
- No schema validation beyond JSON parsing (this layer repairs syntax only).

## Inputs
The interceptor operates on a single string input:
- `raw`: an arbitrary string that may contain:
  - a JSON object/array
  - a JSON object/array embedded in surrounding text
  - malformed JSON that is “near-valid” and can be deterministically repaired

The interceptor MUST NOT mutate outer/provider response dictionaries or their keys. It only consumes and returns strings/parsed JSON values and an explicit report. This preserves the provider/pipeline annotation boundary (D-011).

## Outputs
The interceptor returns a repair result with:
- `ok`: boolean indicating whether a JSON value could be produced
- `value`: parsed JSON value when `ok==True` (object/array/string/number/bool/null)
- `raw`: original input string (unchanged)
- `repaired`: the repaired JSON string when changes were made, otherwise `None`
- `changes`: ordered list of changes that were applied
- `error`: error string when `ok==False`, otherwise `None`

## Determinism Requirements
- Given the same `raw` input, outputs MUST be byte-for-byte stable across runs.
- Repair behavior MUST be rule-based, finite, and order-preserving.
- No randomness, timestamps, environment-dependent parsing, or locale-sensitive behavior.

## Repair Rules (Allowed)
Repairs are attempted in the following order. The interceptor MUST stop at the first rule-sequence that yields valid JSON.

1) **Parse as-is**
   - Attempt `json.loads(raw)`; if it succeeds:
     - `ok=True`
     - `repaired=None`
     - `changes=[]`

2) **Trim leading/trailing whitespace**
   - Apply `strip()` to `raw`.
   - Re-parse.
   - If this changes the string and parsing succeeds, record change:
     - `{"op":"trim_whitespace"}`

3) **Extract JSON substring**
   - If parsing still fails, attempt deterministic extraction:
     - Prefer the earliest `{` and the last matching `}` span (greedy end).
     - If no `{...}` candidate parses, try the earliest `[` and last matching `]`.
   - Re-parse the extracted candidate.
   - If successful, record change:
     - `{"op":"extract_json_substring","kind":"object"|"array"}`

4) **Remove trailing commas**
   - On the current candidate string, remove any trailing comma immediately before `}` or `]`.
   - Re-parse.
   - If successful and a removal occurred, record change:
     - `{"op":"remove_trailing_commas"}`

## Repair Rules (Explicitly Not Allowed)
The interceptor MUST NOT:
- Convert single quotes to double quotes.
- Add/remove braces/brackets.
- Add quotes to unquoted keys.
- Coerce numbers/booleans/strings.
- Re-serialize the parsed value into a canonicalized JSON string (repaired JSON is a minimally edited version of the candidate, not a reformat).

Rationale: these transformations are ambiguous and can silently change meaning.

## Failure Behavior
If no allowed rule-sequence yields a parseable JSON value:
- `ok=False`
- `value=None`
- `repaired=None`
- `changes` includes any attempted deterministic preprocessing that was applied before the final parse attempt (e.g., trim/extract/trailing-comma removal), in order
- `error` is a short, stable string describing the final parse failure class (not a stack trace)

## Report Shape (JSON-serializable)
The result MUST be JSON-serializable (for logging/telemetry) and MUST NOT include stack traces.

Example (successful repair):
```json
{
  "ok": true,
  "raw": "Here you go: {\"a\": 1,}",
  "repaired": "{\"a\": 1}",
  "changes": [
    {"op": "extract_json_substring", "kind": "object"},
    {"op": "remove_trailing_commas"}
  ],
  "value": {"a": 1},
  "error": null
}
```

Example (no repair needed):
```json
{
  "ok": true,
  "raw": "{\"a\": 1}",
  "repaired": null,
  "changes": [],
  "value": {"a": 1},
  "error": null
}
```

## Contract Test Vectors (Normative)
- Valid JSON remains unchanged (no `repaired`, no `changes`).
- Embedded JSON is extracted and parsed; report includes `extract_json_substring`.
- Trailing commas in objects/arrays are removed; report includes `remove_trailing_commas`.
- Inputs requiring disallowed repairs (e.g., single quotes) MUST fail with `ok=False`.
